keepalived双主互为热备

上一篇讲的是主从热备，在任一一个时间段内，只有一台服务器在对外提供服务，这篇介绍双主模式，就是多台服务器可同时对外提供服务器，当其中一台服务器宕机后，将把所有的请求转移到对应的backup机器上。流程图如下：

ip master ip backup ip 代指
192.168.237.130 192.168.237.140 192.168.237.141 A
192.168.237.128 192.168.237.141 192.168.237.140 B

在A机器上配置keepalived 配置文件如下：

root@pz:/usr/local/keepalived/etc/keepalived# vim keepalived_double_master.conf
vrrp_instance VI_1 {
  state MASTER
  interface ens33
  virtual_router_id 50
  nopreempt
  priority 100
  advert_int 1
  authentication {
  auth_type PASS
  ahth_pass 123
  }
  virtual_ipaddress {
  192.168.237.140
  }
}
vrrp_instance VI_2 {
  state BACKUP
  interface ens33
  virtual_router_id 60
  nopreempt
  priority 80
  advert_int 1
  authentication {
  auth_type PASS
  ahth_pass 123
  }
  virtual_ipaddress {
  192.168.237.141
  }
}

在b机器上配置keepalived 配置文件如下

root@pz:/usr/local/keepalived/etc/keepalived# vim keepalived_double_master.conf
vrrp_instance VI_1 {
  state MASTER
  interface ens33
  virtual_router_id 60

  priority 100
  advert_int 1

  auth_type PASS
  ahth_pass 123
  }

  }

vrrp_instance VI_2 {
  state BACKUP
  interface ens33
  virtual_router_id 50
  nopreempt
  priority 80
  advert_int 1

  auth_type PASS

启动

root@pz:/usr/local/keepalived/etc/keepalived# /usr/local/keepalived/sbin/keepalived -D -l -f /usr/local/keepalived/etc/keepalived/keepalived_double_master.conf

分半访问http://192.168.237.140/，http://192.168.237.141/，分别看到，master和backup字样, 杀掉a机器上keepalived进程，再次访问http://192.168.237.140,看到backup字样，启动a机器keepalived进程，关掉b机器keepalived进程，访问http://192.168.237.141，会看到master字样。
