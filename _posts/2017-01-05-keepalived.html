keepalived高可用

keepalived其实主要有两大功能，一：高可用　二：负载均衡。目前一般用来做高可用，也可配合lvs完成集群＋高可用＋故障隔离。本文主要讲如何使用keepalived完成高可用。keepalived之所以能做到高可用，主要是使用了所谓的ippr。keepalived在搭建高可用时，有两种模式，一:主从热备　二：双主热备。先讲主从热备，大概的拓扑图如下

也就是说，虽然有两台服务器，但实际上，同一时间，只有一台是对外服务的，对外服务的服务器有故障时，则虚拟ip切换到另一台，这样就可用继续提供服务了。为了快捷方便，这里以nginx为例。所需主机及ip如下： master:192.168.237.130 bakcup:192.168.237.128 vip:192.168.237.140 配置master的keepalived。配置如下

vrrp_script check_nginx {
script "/tmp/nginx_status.sh" ###监控脚本
interval 1 ###监控时间间隔
}
vrrp_instance nginx{
state MASTER ### 设置为 主
interface eth0 ### 监控网卡
virtual_router_id 50 ### 这个两台服务器必须一样
priority 100 ### 权重值 MASTRE 一定要高于 BAUCKUP
authentication {
auth_type PASS ### 加密方式
auth_pass 123456 ### 加密的密码，两台服务器一定要一样
}
track_script {
check_nginx ### 执行监控的服务
}
virtual_ipaddress {
192.168.237.140 ### VIP地址
}
}

master、backup两台服务器都编写监控脚本，监控脚本可以按需求写，０表示没有正常，１表示有故障

#vim /tmp/nginx_status.sh
#!/bin/bash
count = `ps aux | grep nginx | wc -l`
if [ $count > 1 ]; then
  exit 0
else
  exit 1
fi

然后是backup的配置

vrrp_script check_nginx {
script "/tmp/nginx_status.sh" ###监控脚本
interval 1 ###监控时间间隔
}
vrrp_instance nginx{
state BACKUP ### 设置为从
interface eth0 ### 监控网卡
virtual_router_id 50 ### 这个两台服务器必须一样
priority 80 ### 权重值,该值不能超过master的权重值
authentication {
auth_type PASS ### 加密方式
auth_pass 123456 ### 加密的密码，两台服务器一定要一样
}
track_script {
check_nginx ### 执行监控的服务
}
virtual_ipaddress {
192.168.237.140 ### VIP地址
}
}

分别在master、backup中执行

#echo 'master'>/var/www/index.html
#echo 'backup'>/var/www/index.html
